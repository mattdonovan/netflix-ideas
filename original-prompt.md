# Original prompt — project genesis

This is the first prompt Matt sent to kick off the netflix-ideas project, preserved verbatim (typos and all) for future writeups.

---

I want to create a protoype of an idea I have for netflix. The execution of the idea needs to include excellent, highly polished UI design - far tight that your design skills can accomplish on a first pass. So in addition to the creating the prototoype, I want you to create a set of design review lenses based on '/Users/mattdonovan/Desktop/Projects/ups-momentum/context/review-lenses.md' - but crafted for this project. Research designers like Josh Mateo (https://joshmateo.com/), Nyati Gupta (https://www.linkedin.com/in/niygup/), and the Netflix Tech Blog (https://netflixtechblog.com/). I will also need educational guidance and review lenses on designing for TV as a surface. I've never done that before.

The prototype idea:

I want to protoype two netflix feature ideas. The first is to solve the redundant content problem. Netflix has near ininite content, but the recommendation engine pigeon holes you into really specific niches. I desire to discover totally new stuff and want to replace the predefinded content groups with my own "channels". I imagine doing this by focusing on a content row title and speaking into my remote the kinds of stuff I'd like to see there instead. Maybe it has some "popular" creations. But on the backend, it takes whatever I say (or possible type) and uses AI to assign a category or create a new one on the fly and pull in content using Netflix's Ranker (https://netflixtechblog.com/optimizing-recommendation-systems-with-jdks-vector-api-30d2830401ec). We won't really do that, but we'll communicate that's the idea (unless we can really do that?). I could say "indie movies from this year" or "monster movies with soul" or "stuff teenagers are watching" or "movies with warm cinemetography" or "new nostalgia", and the AI would fill the category, and ask me what I think. I can tweak it endlessly.

The second idea is to invite people to netflix through a show, movie, podcast, or game (I don't know the game space very well). The idea is, while watching an episode I notice an "Share" affordance. Clicking it reveals a screen/modal/whatever that tells me I have 3 invites this month and presents a QR code to scan. Scanning the QR code opens a webpage on my phone tited to my netflix account (automaticall logged in). I see I have three invites. The CTA is to share this show or movie with a friend. I copy a link an text it/email it/snap chat it/whatever and they get access to watch the first 3 episodes or one whole movie (depending on what I shared) for free with a uniqe one-time code. I get notifed when they sign up and can share selections from my viewing history I think they'll like. Netflix then shows them a content row in their app of "Matt's Recommendations for You". Unless it's the show/movie/game I invited them to, they'll need to subscribe to watch it. Potentially show some "free" netflix content for these users to increase usage - or let them watch for free for a week.

I want you to plan this whole project. We should use material UI because that's what Hawkins (netflix's design system) is funamentally built on. I don't want to copy Hawkins - but instead roll a wireframe version of material ui that uses Hawkins spacing, typography, layout, interactions, etc. Do as much research as you can on Hawkins. It's not super well documented from what I can find, but I'd like to stand on as much foundational knoweldge as we can.

We'll need to create a github repo we can push to so the review lenses can post to it per the file instrcutions.

Assess this plan and tell me what your plan is before you start to build. I'd like to get you set up to make an entire first pass at one of the prototypes, run the review lenses, and post the review to github along with committing and pushing the code. I think I would also like to build into our process and UI an "experiments" section where I can view mutliple ideas side--by-side with a simple prompt without affecting the main prototype.
